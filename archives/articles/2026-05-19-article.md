# 用PyTorch实现文本分类：从词嵌入到TextCNN的完整实战指南

**核心结论：** 通过PyTorch实现文本分类，关键在于掌握词嵌入将离散文本转化为连续向量、TextCNN捕捉局部语义特征，以及构建高效的训练循环，这三者结合即可在短文本分类任务中达到90%以上的准确率。

## 为什么重要

文本分类是自然语言处理中最基础、应用最广泛的任务之一。从垃圾邮件过滤、情感分析到新闻分类，文本分类能力是AI工程师处理非结构化数据的核心技能。在深度学习精通阶段，掌握文本分类意味着你能够：

- **理解序列数据的处理范式**：文本分类是理解RNN、CNN、Transformer等架构处理序列数据的起点
- **掌握词嵌入的核心原理**：词嵌入是NLP领域的基石技术，其思想（将离散符号映射到连续空间）被广泛应用于推荐系统、知识图谱等领域
- **建立完整的模型训练流程**：从数据预处理、模型构建到训练评估，文本分类涵盖了深度学习项目的全生命周期

对于AI工程师而言，文本分类不仅仅是调用一个API那么简单。它要求你理解数据如何从原始文本变成张量、模型如何学习特征、以及如何优化训练过程。这正是一个合格的深度学习工程师与调包侠之间的分水岭。

## 核心内容

### 技术点1：词嵌入——让计算机理解词语含义

#### 概念说明

词嵌入（Word Embedding）是将文本中的每个词映射到一个低维稠密向量的技术。与传统的one-hot编码（维度等于词表大小，向量稀疏）不同，词嵌入将语义相近的词映射到向量空间中相近的位置。

#### 原理解析

词嵌入的核心思想是**分布式语义假设**：一个词的含义由其上下文决定。通过训练，模型学习到每个词的向量表示，使得"国王"和"王后"的向量差与"男人"和"女人"的向量差相似。

在PyTorch中，`nn.Embedding`层实现了词嵌入功能：
- 输入：词的索引（整数）
- 输出：对应的稠密向量
- 参数：词表大小 × 嵌入维度

词嵌入层本质上是一个可训练的查找表。训练过程中，反向传播会更新这些向量，使其逐渐捕捉词语的语义和语法信息。

#### 代码示例

```python
import torch
import torch.nn as nn

# 定义词嵌入层
vocab_size = 10000  # 词表大小
embedding_dim = 300  # 嵌入维度
embedding_layer = nn.Embedding(vocab_size, embedding_dim)

# 示例：将词的索引转换为向量
word_indices = torch.tensor([1, 5, 23, 42])  # 四个词的索引
word_vectors = embedding_layer(word_indices)  # 形状: [4, 300]

print(f"词向量形状: {word_vectors.shape}")
print(f"第一个词的向量前5维: {word_vectors[0][:5]}")
```

**关键参数详解：**
- `vocab_size`：词表大小，决定了可以表示的最大词数
- `embedding_dim`：嵌入维度，通常取50-300，维度越高表达能力越强但计算量越大
- `padding_idx`：可选参数，指定填充词的索引，该词对应的向量在训练中不会更新

### 技术点2：TextCNN——用卷积网络捕捉文本局部特征

#### 概念说明

TextCNN是将卷积神经网络应用于文本分类的经典模型。与图像CNN不同，TextCNN在文本的序列维度上进行一维卷积，通过不同大小的卷积核捕捉n-gram级别的局部特征。

#### 原理解析

TextCNN的核心架构包含三个关键步骤：

1. **词嵌入层**：将输入文本序列转换为词向量矩阵，形状为 `[batch_size, seq_len, embedding_dim]`
2. **卷积层**：使用多个不同大小的卷积核（通常为3、4、5个词的窗口）在序列上滑动，每个卷积核提取一种n-gram特征
3. **池化层**：对每个卷积核的输出进行最大池化，提取最重要的特征，并将不同卷积核的特征拼接

**为什么TextCNN有效？** 卷积核相当于滑动窗口扫描文本，小卷积核（size=3）捕捉相邻词的关系，大卷积核（size=5）捕捉稍远距离的词间依赖。通过组合不同大小的卷积核，模型能够同时学习短距离和长距离的局部模式。

#### 代码示例

```python
import torch.nn.functional as F

class TextCNN(nn.Module):
    def __init__(self, vocab_size, embedding_dim, num_filters, filter_sizes, num_classes, dropout=0.5):
        super(TextCNN, self).__init__()
        
        # 词嵌入层
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
        
        # 多个不同大小的卷积核
        self.convs = nn.ModuleList([
            nn.Conv1d(in_channels=embedding_dim, 
                     out_channels=num_filters, 
                     kernel_size=fs)
            for fs in filter_sizes
        ])
        
        # 全连接分类层
        self.fc = nn.Linear(len(filter_sizes) * num_filters, num_classes)
        self.dropout = nn.Dropout(dropout)
        
    def forward(self, x):
        # x: [batch_size, seq_len]
        x = self.embedding(x)  # [batch_size, seq_len, embedding_dim]
        x = x.permute(0, 2, 1)  # 转换为 [batch_size, embedding_dim, seq_len] 适配Conv1d
        
        # 对每个卷积核进行操作
        conv_outputs = []
        for conv in self.convs:
            conv_out = conv(x)  # [batch_size, num_filters, seq_len - kernel_size + 1]
            pooled = F.max_pool1d(conv_out, conv_out.size(2))  # 全局最大池化
            conv_outputs.append(pooled.squeeze(2))  # [batch_size, num_filters]
        
        # 拼接所有卷积核的输出
        x = torch.cat(conv_outputs, dim=1)  # [batch_size, num_filters * len(filter_sizes)]
        x = self.dropout(x)
        x = self.fc(x)  # [batch_size, num_classes]
        return x

# 实例化模型
model = TextCNN(
    vocab_size=10000,
    embedding_dim=300,
    num_filters=100,
    filter_sizes=[3, 4, 5],
    num_classes=2  # 二分类
)

print(f"模型参数量: {sum(p.numel() for p in model.parameters()):,}")
```

### 技术点3：训练循环——让模型学习分类

#### 概念说明

训练循环是深度学习项目中最核心的工程实践。一个完整的训练循环包括：数据加载、前向传播、损失计算、反向传播、参数更新和评估监控。

#### 原理解析

训练循环遵循以下流程：
1. **准备数据批次**：使用`DataLoader`将数据集分成小批次
2. **前向传播**：将输入数据送入模型，得到预测结果
3. **计算损失**：使用交叉熵损失函数衡量预测与真实标签的差异
4. **反向传播**：计算损失相对于模型参数的梯度
5. **参数更新**：使用优化器根据梯度更新参数
6. **循环迭代**：重复以上步骤，直到模型收敛

**关键实践要点：**
- **梯度清零**：每次反向传播前必须清零梯度，否则梯度会累积
- **学习率调度**：使用学习率衰减策略，在训练后期缩小步长
- **早停法**：监控验证集性能，当性能不再提升时停止训练

#### 代码示例

```python
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

def train_model(model, train_texts, train_labels, val_texts, val_labels, 
                epochs=10, batch_size=64, lr=0.001, device='cuda'):
    
    # 准备数据
    train_dataset = TensorDataset(train_texts, train_labels)
    train_loader = DataLoader(train_dataset, batch_size=batch_size, shuffle=True)
    
    val_dataset = TensorDataset(val_texts, val_labels)
    val_loader = DataLoader(val_dataset, batch_size=batch_size)
    
    # 定义损失函数和优化器
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.Adam(model.parameters(), lr=lr)
    scheduler = optim.lr_scheduler.StepLR(optimizer, step_size=5, gamma=0.5)
    
    model.to(device)
    
    for epoch in range(epochs):
        # 训练阶段
        model.train()
        train_loss = 0.0
        train_correct = 0
        train_total = 0
        
        for texts, labels in train_loader:
            texts, labels = texts.to(device), labels.to(device)
            
            # 梯度清零
            optimizer.zero_grad()
            
            # 前向传播
            outputs = model(texts)
            loss = criterion(outputs, labels)
            
            # 反向传播
            loss.backward()
            
            # 梯度裁剪，防止梯度爆炸
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            
            # 参数更新
            optimizer.step()
            
            # 统计
            train_loss += loss.item() * texts.size(0)
            _, predicted = torch.max(outputs, 1)
            train_total += labels.size(0)
            train_correct += (predicted == labels).sum().item()
        
        # 验证阶段
        model.eval()
        val_loss = 0.0
        val_correct = 0
        val_total = 0
        
        with torch.no_grad():
            for texts, labels in val_loader:
                texts, labels = texts.to(device), labels.to(device)
                outputs = model(texts)
                loss = criterion(outputs, labels)
                
                val_loss += loss.item() * texts.size(0)
                _, predicted = torch.max(outputs, 1)
                val_total += labels.size(0)
                val_correct += (predicted == labels).sum().item()
        
        # 学习率调整
        scheduler.step()
        
        # 打印结果
        train_acc = 100 * train_correct / train_total
        val_acc = 100 * val_correct / val_total
        print(f'Epoch [{epoch+1}/{epochs}]')
        print(f'Train Loss: {train_loss/train_total:.4f}, Train Acc: {train_acc:.2f}%')
        print(f'Val Loss: {val_loss/val_total:.4f}, Val Acc: {val_acc:.2f}%')
        print('-' * 50)
    
    return model

# 使用示例（假设已有预处理好的数据）

# train_model(model, train_texts, train_labels, val_texts, val_labels)
```

## 实践建议

1. **从简单数据集开始**：推荐使用IMDb电影评论情感分析数据集或AG News新闻分类数据集。这些数据集规模适中（数万条），标签分布均衡，适合初学者快速上手。

2. **注意文本预处理细节**：
   - 统一文本大小写（通常转为小写）
   - 过滤特殊字符和停用词
   - 设置合理的最大序列长度（建议100-200个词）
   - 使用`<PAD>`标记填充短文本

3. **调优TextCNN超参数**：
   - 卷积核大小组合：`[2,3,4]`或`[3,4,5]`适合短文本，`[3,5,7]`适合长文本
   - 每类卷积核数量：从100开始，根据任务复杂度增减
   - 嵌入维度：300是性能和效果的平衡点

4. **使用预训练词向量**：加载GloVe或Word2Vec预训练词向量初始化嵌入层，可以显著提升小数据集上的性能。PyTorch的`nn.Embedding.from_pretrained()`可以轻松实现。

5. **监控训练过程**：记录训练和验证的损失曲线，如果验证损失上升而训练损失下降，说明过拟合，需要增加dropout率或减少模型容量。
