# 词向量：从Word2Vec到Contextual Embedding

## 核心结论

**词向量的核心演进是从静态、无上下文的离散表示，走向动态、上下文感知的连续表示，这一转变是自然语言处理从“特征工程”迈向“表示学习”的标志性跨越。**

## 为什么重要

在深度学习介入自然语言处理之前，机器理解语言的方式极其原始。**One-hot编码**将每个词视为高维稀疏向量，不仅维度灾难严重，更致命的是无法表达词与词之间的语义相似性——“猫”和“狗”在向量空间中没有任何关联。这种孤立表示成为NLP性能提升的根本瓶颈。

词向量的出现彻底改变了这一局面。通过将词语映射到低维连续向量空间，模型能够自动学习到词之间的语义和语法关系。**Word2Vec的Skip-gram和CBOW模型，以极其简洁的架构证明了“通过上下文学习语义”这一朴素思想的巨大威力。** 这不仅是深度学习在NLP领域的第一次重大胜利，更为后续所有预训练语言模型奠定了基础。

理解词向量，意味着你掌握了NLP从“规则驱动”到“数据驱动”跃迁的核心密码。**从Word2Vec到ELMo再到BERT的演进路径，本质上就是模型对语言上下文信息的利用从“局部窗口”走向“全局语境”的过程。** 对于AI工程师而言，这不仅是技术知识，更是一种思维范式的转换——从设计特征到让模型学习特征。

## 核心内容

### 一、静态词向量：Word2Vec

Word2Vec由Mikolov等人在2013年提出，包含两种核心架构：**Skip-gram**和**CBOW**。其核心思想是**分布式假设**：一个词的含义由其上下文决定。

#### 1.1 Skip-gram：由中心词预测上下文

**概念说明**

Skip-gram模型的任务是：给定一个中心词，预测它周围的上下文词。例如在句子“我 爱 自然 语言 处理”中，如果窗口大小为2，以“自然”为中心词，模型需要预测“爱”、“语言”和“处理”这些上下文词。

**原理解析**

模型结构极其简洁：
- **输入层**：中心词（one-hot编码，维度V）
- **投影层**：将one-hot向量映射到d维稠密向量
- **输出层**：通过softmax输出V维概率分布，表示每个词作为上下文的概率

**关键创新**：通过**负采样（Negative Sampling）**解决softmax计算量过大的问题。负采样的核心思想是：将多分类问题转化为二分类问题——判断一个词对是否来自真实上下文。

**代码示例**

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SkipGram(nn.Module):
    def __init__(self, vocab_size, embedding_dim):
        super(SkipGram, self).__init__()
        self.embeddings = nn.Embedding(vocab_size, embedding_dim)
        
    def forward(self, center_words, context_words, negative_words):
        """
        center_words: [batch_size]
        context_words: [batch_size]
        negative_words: [batch_size, neg_count]
        """
        # 获取中心词向量
        center_emb = self.embeddings(center_words)  # [batch, dim]
        
        # 正样本得分
        pos_emb = self.embeddings(context_words)  # [batch, dim]
        pos_score = torch.sum(center_emb * pos_emb, dim=1)  # [batch]
        pos_loss = -F.logsigmoid(pos_score)
        
        # 负样本得分
        neg_emb = self.embeddings(negative_words)  # [batch, neg, dim]
        # 扩展中心词向量维度
        center_emb_expanded = center_emb.unsqueeze(1)  # [batch, 1, dim]
        neg_score = torch.sum(center_emb_expanded * neg_emb, dim=2)  # [batch, neg]
        neg_loss = -F.logsigmoid(-neg_score).sum(dim=1)  # [batch]
        
        return (pos_loss + neg_loss).mean()
```

#### 1.2 CBOW：由上下文预测中心词

**概念说明**

CBOW与Skip-gram相反：给定上下文词，预测中心词。继续使用上面的例子，输入“爱”、“语言”、“处理”，预测中心词“自然”。

**原理解析**

CBOW的架构与Skip-gram对称：
- **输入层**：多个上下文词的one-hot向量
- **投影层**：将所有上下文词向量相加（或取平均）
- **输出层**：预测中心词

**关键区别**：CBOW对上下文词进行聚合操作，因此训练速度更快，但对低频词的表示能力不如Skip-gram。**在实践中，Skip-gram在处理小型数据集和罕见词时表现更好，CBOW则在大规模语料上训练更快。**

**实践指导**

```python
class CBOW(nn.Module):
    def __init__(self, vocab_size, embedding_dim, context_size):
        super(CBOW, self).__init__()
        self.embeddings = nn.Embedding(vocab_size, embedding_dim)
        self.linear = nn.Linear(embedding_dim, vocab_size)
        self.context_size = context_size
        
    def forward(self, context_words):
        # context_words: [batch_size, context_size]
        emb = self.embeddings(context_words)  # [batch, context, dim]
        emb = emb.mean(dim=1)  # [batch, dim]
        out = self.linear(emb)  # [batch, vocab]
        return out
```

### 二、动态词向量：ELMo

**概念说明**

Word2Vec的致命缺陷是**静态性**：每个词只有一个固定的向量表示。但在真实语言中，词义高度依赖上下文。例如“苹果”在“吃苹果”和“苹果公司”中含义完全不同。**ELMo（Embeddings from Language Models）**通过双向语言模型解决了这个问题。

**原理解析**

ELMo的核心创新点：
1. **双向LSTM**：使用前向和后向两个语言模型，捕获单词的左右上下文信息
2. **多层表示融合**：不同层的LSTM捕获不同粒度的语言特征（底层偏向语法，高层偏向语义）
3. **动态权重**：最终表示是各层表示的加权组合，权重由任务学习得到

**关键洞察**：ELMo的表示是**函数式的**，而非静态查找表。对于同一个词，它在不同句子中的向量表示不同。

**代码示例**

```python
import torch
import torch.nn as nn

class ELMo(nn.Module):
    def __init__(self, vocab_size, embedding_dim, hidden_dim, num_layers=2):
        super(ELMo, self).__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
        
        # 前向LSTM
        self.forward_lstm = nn.LSTM(
            embedding_dim, hidden_dim, num_layers, 
            batch_first=True, bidirectional=False
        )
        
        # 后向LSTM
        self.backward_lstm = nn.LSTM(
            embedding_dim, hidden_dim, num_layers,
            batch_first=True, bidirectional=False
        )
        
        # 可学习的权重参数
        self.layer_weights = nn.Parameter(torch.ones(num_layers * 2 + 1) / (num_layers * 2 + 1))
        
    def forward(self, x):
        # x: [batch, seq_len]
        emb = self.embedding(x)  # [batch, seq, dim]
        
        # 前向LSTM
        forward_out, _ = self.forward_lstm(emb)
        
        # 后向LSTM（输入反转）
        backward_emb = torch.flip(emb, dims=[1])
        backward_out, _ = self.backward_lstm(backward_emb)
        backward_out = torch.flip(backward_out, dims=[1])
        
        # 获取每层表示（简化版，实际需要每层输出）
        # 这里仅演示最后一层
        all_layers = [emb, forward_out, backward_out]
        
        # 加权融合
        weights = F.softmax(self.layer_weights, dim=0)
        weighted_sum = sum(w * layer for w, layer in zip(weights, all_layers))
        
        return weighted_sum  # [batch, seq, hidden_dim]
```

**实践指导**：使用ELMo时，**通常将预训练好的ELMo作为特征提取器**，将其输出与下游任务的输入拼接。ELMo的引入使得许多NLP任务取得了显著的性能提升，尤其是在标注任务（如命名实体识别、词性标注）上。

### 三、从静态到动态：技术演进的内在逻辑

理解这一演进，需要把握三个关键维度：

1. **上下文范围**：Word2Vec使用固定窗口（通常2-5个词），ELMo使用整个句子，BERT更进一步使用双向Transformer
2. **表示灵活性**：从“一个词一个向量”到“一个词在不同语境中有不同向量”
3. **学习目标**：从简单的“预测邻居词”到复杂的“语言模型预训练”

**这一演进的本质是模型对语言复杂性的建模能力不断提升。** Word2Vec捕获了词之间的共现关系，ELMo捕获了语境化的语义，而后续的BERT则进一步捕获了深层的双向上下文信息。

## 知识依赖说明

理解本文内容需要以下前置知识：

- **无监督学习基础**：Word2Vec的训练本质上是一种无监督学习——不需要标注数据，仅利用文本中的共现信息。理解无监督学习的基本概念（如聚类、降维）有助于把握词向量的思想精髓。
- **神经网络基础**：需要理解前向传播、反向传播、梯度下降等基本概念。Skip-gram和CBOW虽然结构简单，但包含完整的神经网络训练流程。
- **概率论基础**：softmax函数、负采样中的概率分布、对数似然等概念会频繁出现。
- **NLP基本概念**：词袋模型、one-hot编码、n-gram等传统NLP方法作为对比基础。

如果你对上述知识不熟悉，建议先回顾阶段2「机器学习核心」中的无监督学习部分，以及阶段3前期的神经网络基础内容。

## 实践建议

1. **从Gensim开始快速体验**：使用Gensim库训练Word2Vec只需几行代码。先在小规模语料（如维基百科子集）上跑通流程，观察词向量的语义相似性（如“king - man + woman ≈ queen”）。

2. **手动实现负采样**：不要依赖框架的自动实现，自己写一遍负采样逻辑。这能让你深刻理解Word2Vec的核心优化技巧，以及它在处理大规模词汇表时的工程智慧。

3. **比较不同上下文窗口的影响**：在相同语料上，分别用窗口大小为2、5、10训练Word2Vec，观察词向量在语法和语义上的差异。你会发现小窗口更偏向语法关系，大窗口更偏向语义主题。

4. **在具体任务中对比静态与动态表示**：选择一个序列标注任务（如命名实体识别），分别使用Word2Vec和ELMo作为输入特征，观察性能差异。**数据量越小，动态表示的优势越明显。**

5. **可视化词向量空间**：使用t-SNE或PCA将高维词向量降到2D，观察语义相近的词在空间中的聚类情况。这是理解词向量质量最直观的方法。
