# 检索增强生成的进阶技巧：混合检索、重排序与查询改写

## 核心结论

**通过混合检索、重排序和查询改写三个进阶技巧，可以将RAG系统的检索准确率从60%提升至90%以上，这是从“能用”到“好用”的关键跨越。**

## 为什么重要

在阶段4「大模型应用开发」中，RAG基础让你掌握了“检索+生成”的基本框架：将用户问题转化为向量，在知识库中检索相似片段，然后让大模型基于这些片段生成答案。然而，在实际生产环境中，你很快会发现基础RAG面临着三个致命问题：**检索召回率低**（单一向量检索无法捕捉语义差异）、**排序质量差**（最相关文档可能不在前三）、**用户意图模糊**（原始查询与知识库表述不匹配）。

这三个问题直接导致大模型基于错误或无关信息生成答案，产生“幻觉”或“答非所问”。作为AI工程师，你需要从“能搭建RAG系统”进化到“能优化RAG系统”，这正是进阶技巧的价值所在。

本文将深入讲解三个核心技术点：**混合检索**（结合向量与关键词）、**重排序**（精排候选文档）、**查询改写**（优化用户输入）。掌握这些技巧后，你将能设计出检索准确率超过90%的RAG系统，真正满足企业级应用的需求。

## 核心内容

### 混合检索：向量与关键词的双重保障

#### 概念说明

混合检索指**同时使用向量检索和关键词检索（如BM25）**，并将两者结果融合。向量检索擅长捕捉语义相似性（例如“如何预防感冒”与“增强免疫力的方法”），而关键词检索擅长精确匹配专业术语（例如“Transformer架构”必须精确匹配）。两者互补，能显著提升召回率。

#### 原理解析

混合检索的核心是**分数融合**。假设向量检索返回文档列表A（每篇有相似度分数），关键词检索返回文档列表B（每篇有BM25分数），你需要将它们归一化到同一量级，然后加权求和。

常用方法：
1. **Min-Max归一化**：将分数缩放到[0,1]区间
2. **倒数秩融合（RRF）**：`score = 1/(k + rank)`，避免分数尺度问题
3. **加权平均**：`final_score = α * vector_score + (1-α) * keyword_score`，其中α通常取0.3-0.7

#### 代码示例

```python
import numpy as np
from rank_bm25 import BM25Okapi
from sentence_transformers import SentenceTransformer
from typing import List, Dict

class HybridRetriever:
    def __init__(self, documents: List[str], alpha: float = 0.5):
        """
        alpha: 向量检索权重，(1-alpha)为关键词检索权重
        """
        self.documents = documents
        self.alpha = alpha
        
        # 初始化向量检索模型
        self.encoder = SentenceTransformer('all-MiniLM-L6-v2')
        self.doc_embeddings = self.encoder.encode(documents, normalize_embeddings=True)
        
        # 初始化BM25（关键词检索）
        tokenized_corpus = [doc.split() for doc in documents]
        self.bm25 = BM25Okapi(tokenized_corpus)
    
    def retrieve(self, query: str, top_k: int = 10) -> List[Dict]:
        # 1. 向量检索
        query_embedding = self.encoder.encode(query, normalize_embeddings=True)
        vector_scores = np.dot(self.doc_embeddings, query_embedding)
        # Min-Max归一化
        vector_scores = (vector_scores - vector_scores.min()) / (vector_scores.max() - vector_scores.min() + 1e-8)
        
        # 2. 关键词检索
        tokenized_query = query.split()
        bm25_scores = self.bm25.get_scores(tokenized_query)
        # Min-Max归一化
        bm25_scores = (bm25_scores - bm25_scores.min()) / (bm25_scores.max() - bm25_scores.min() + 1e-8)
        
        # 3. 融合分数
        final_scores = self.alpha * vector_scores + (1 - self.alpha) * bm25_scores
        
        # 4. 获取Top-K
        top_indices = np.argsort(final_scores)[::-1][:top_k]
        return [
            {
                "doc": self.documents[i],
                "score": final_scores[i],
                "vector_score": vector_scores[i],
                "bm25_score": bm25_scores[i]
            }
            for i in top_indices
        ]

# 使用示例
documents = [
    "Transformer模型使用自注意力机制处理序列数据",
    "BERT是一种基于Transformer的预训练语言模型",
    "自注意力机制允许模型关注输入序列的所有位置"
]
retriever = HybridRetriever(documents, alpha=0.6)
results = retriever.retrieve("什么是自注意力机制")
for r in results:
    print(f"文档: {r['doc']}, 融合分数: {r['score']:.3f}")
```

**实践要点**：
- α值需要通过验证集调优，一般从0.5开始尝试
- 对于专业领域（如医疗、法律），可适当提高关键词检索权重（降低α）
- 大规模场景下，建议使用Elasticsearch实现BM25检索

### 重排序：从粗筛到精排

#### 概念说明

重排序是在混合检索之后，**使用更精细的模型对候选文档重新排序**。混合检索可能返回50篇文档，但大模型上下文有限，只能处理3-5篇。重排序的任务就是从中选出最相关的几篇，**大幅提升最终答案质量**。

#### 原理解析

重排序的核心思想是**交叉编码（Cross-Encoder）**。与双编码器（将查询和文档分别编码后计算余弦相似度）不同，交叉编码将查询和文档拼接后输入一个Transformer模型，直接输出相关性分数。这种方式能捕捉查询与文档间的深层交互，但计算量更大（所以只对Top-K文档重排序）。

流程：
1. 粗召回（混合检索）返回50篇候选文档
2. 将这50篇文档与查询拼接，输入交叉编码器
3. 交叉编码器输出每对的相关性分数
4. 按分数降序排列，取Top-3输入大模型

#### 代码示例

```python
from sentence_transformers import CrossEncoder
import torch

class Reranker:
    def __init__(self, model_name: str = 'cross-encoder/ms-marco-MiniLM-L-6-v2'):
        """
        使用预训练的交叉编码模型
        """
        self.model = CrossEncoder(model_name, max_length=512)
    
    def rerank(self, query: str, candidates: List[Dict], top_k: int = 3) -> List[Dict]:
        # 准备输入对
        pairs = [(query, cand["doc"]) for cand in candidates]
        
        # 计算相关性分数
        scores = self.model.predict(pairs, show_progress_bar=False)
        
        # 更新候选文档的分数
        for i, cand in enumerate(candidates):
            cand["rerank_score"] = float(scores[i])
        
        # 按重排序分数降序排列
        candidates.sort(key=lambda x: x["rerank_score"], reverse=True)
        
        return candidates[:top_k]

# 完整RAG流程示例
def advanced_rag_pipeline(query: str, retriever: HybridRetriever, reranker: Reranker, llm):
    # 1. 混合检索（粗召回）
    candidates = retriever.retrieve(query, top_k=50)
    
    # 2. 重排序（精排）
    top_candidates = reranker.rerank(query, candidates, top_k=3)
    
    # 3. 构建上下文
    context = "\n\n".join([c["doc"] for c in top_candidates])
    
    # 4. 生成答案
    prompt = f"""基于以下信息回答问题：
    
{context}

问题：{query}
答案："""
    
    return llm.generate(prompt)
```

**实践要点**：
- 交叉编码模型推荐：`ms-marco-MiniLM-L-6-v2`（快速）、`BAAI/bge-reranker-v2-m3`（高精度）
- 重排序的Top-K值建议3-5，太多会稀释大模型注意力
- 对于长文档，需要截断至模型最大长度（通常512 tokens）

### 查询改写：让用户的问题更“懂”知识库

#### 概念说明

查询改写指**在检索前对用户原始查询进行转换**，使其更符合知识库的表述方式。用户可能使用口语化表达（“怎么治感冒”），而知识库使用专业表述（“上呼吸道感染的临床治疗方案”）。查询改写能弥合这一鸿沟。

#### 原理解析

查询改写的三种常用策略：
1. **同义词替换**：将口语词替换为专业术语（“治”->“治疗”，“感冒”->“上呼吸道感染”）
2. **查询扩展**：添加相关概念，扩大召回范围（“感冒”->“感冒 流感 发热 咳嗽”）
3. **意图重写**：用大模型将用户问题重写为更清晰、更完整的查询

其中，**基于大模型的查询改写**效果最好。核心思路是：使用一个小型语言模型，根据用户原始查询和历史对话，生成多个改写版本，然后对每个版本进行检索，最后合并结果。

#### 代码示例

```python
from openai import OpenAI
from typing import List

class QueryRewriter:
    def __init__(self, api_key: str, model: str = "gpt-3.5-turbo"):
        self.client = OpenAI(api_key=api_key)
        self.model = model
    
    def rewrite(self, query: str, num_variants: int = 3) -> List[str]:
        """
        生成多个查询改写版本
        """
        prompt = f"""你是一位AI检索专家。请将用户的问题改写为{num_variants}个不同的版本，要求：
1. 使用更专业、更完整的表述
2. 每个版本侧重点不同（如一个侧重病因，一个侧重治疗）
3. 直接输出改写后的查询，每行一个，不要编号

用户问题：{query}

改写后的查询："""
        
        response = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.7,
            max_tokens=200
        )
        
        rewritten = response.choices[0].message.content.strip().split("\n")
        # 清理并返回
        return [q.strip() for q in rewritten if q.strip() and not q.strip().startswith(("1.", "2.", "3."))]
    
    def multi_query_retrieve(self, query: str, retriever: HybridRetriever, top_k: int = 10) -> List[Dict]:
        """
        多查询检索：对多个改写版本分别检索，合并结果
        """
        # 生成改写版本（包含原始查询）
        queries = [query] + self.rewrite(query, num_variants=2)
        
        # 对每个查询进行检索
        all_results = []
        for q in queries:
            results = retriever.retrieve(q, top_k=top_k)
            all_results.extend(results)
        
        # 去重并保留最高分数
        seen_docs = set()
        final_results = []
        for r in sorted(all_results, key=lambda x: x["score"], reverse=True):
            if r["doc"] not in seen_docs:
                seen_docs.add(r["doc"])
                final_results.append(r)
        
        return final_results[:top_k]

# 使用示例
rewriter = QueryRewriter(api_key="your-api-key")
# 原始查询
original_query = "怎么治感冒"
# 改写后的查询
rewritten_queries = rewriter.rewrite(original_query)
print("改写版本：", rewritten_queries)
# 输出示例：['上呼吸道感染的治疗方案', '感冒的药物治疗与家庭护理']
```

**实践要点**：
- 使用小模型（如GPT-3.5-turbo）进行改写，避免大模型延迟
- 改写版本数量建议2-3个，过多会导致检索结果重复
- 对于多轮对话，改写时应包含历史上下文

2. **文本嵌入基础**：了解Sentence-BERT、双编码器等概念，理解如何将文本转化为向量。
3. **信息检索基础**：了解BM25算法、TF-IDF等传统检索方法，理解关键词检索的原理。
4. **Transformer模型基础**：了解注意力机制、交叉编码与双编码的区别，这对理解重排序模型至关重要。

## 实践建议

1. **从混合检索开始**：这是最容易落地的进阶技巧。先在现有RAG系统中加入BM25检索，使用RRF融合方法，观察召回率提升。建议使用Elasticsearch或Whoosh实现关键词检索。
2. **重排序使用轻量模型**：生产环境中，推荐使用`cross-encoder/ms-marco-MiniLM-L-6-v2`，它在CPU上也能快速推理。对于高精度场景，可以部署`BAAI/bge-reranker-v2-m3`。
3. **查询改写与业务术语表结合**：准备一份领域术语映射表（如“感冒”->“上呼吸道感染”），作为查询改写的辅助工具。这比纯模型改写更稳定。
4. **建立评估指标**：使用MRR（平均倒数排名）和NDCG（归一化折损累计增益）评估检索质量。建议建立100-200条测试用例的评估集，每次改动后运行评估。
5. **逐步叠加，而非一次性全上**：建议按“混合检索→重排序→查询改写”的顺序逐步引入，每次只增加一个技巧，用评估指标验证效果提升。避免一次性改动过多导致问题难以定位。
