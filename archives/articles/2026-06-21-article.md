# 数据不平衡问题处理

## 核心结论

**处理数据不平衡的核心在于理解模型对多数类的偏向性，并通过数据层面或算法层面的干预，迫使模型学习少数类的有效特征，而非简单追求整体准确率。**

## 为什么重要

在真实世界的机器学习应用中，数据不平衡是常态而非例外。无论是信用卡欺诈检测（欺诈交易通常占比<1%）、医疗诊断（罕见病发病率极低）、还是工业缺陷检测（合格产品占绝大多数），模型都面临着“多数类淹没少数类”的困境。如果直接训练，模型会倾向于将所有样本预测为多数类，从而获得看似很高的准确率（如99%），但实际上对少数类毫无识别能力——这正是**准确率陷阱**。

对于AI工程师而言，数据不平衡处理是**从“跑通模型”到“实战可用”的关键跨越**。在阶段2「机器学习核心」中，你已经掌握了模型评估的基本方法（如混淆矩阵、精确率、召回率、F1分数），现在需要进一步学习：当评估指标反映出模型对少数类失效时，如何系统地解决问题。这不仅是调参技巧，更是对模型学习机制理解的深化——**数据分布决定了模型的行为边界**。

## 核心内容

### 过采样：从数据层面增加少数类

#### 概念说明

**过采样（Oversampling）** 是最直观的平衡方法：通过复制少数类样本，使其数量与多数类接近。最简单的实现是**随机过采样**——从少数类中有放回地随机抽取样本，直到达到目标数量。

#### 原理解析

随机过采样的核心逻辑是“给模型更多机会看到少数类”。假设原始数据中多数类有10000个，少数类只有100个，随机过采样将少数类复制到10000个。模型在训练时，每次batch中两类样本比例接近1:1，从而避免了对多数类的过度拟合。

**但随机过采样存在明显缺陷**：它只是复制现有样本，没有引入新信息。这会导致模型对少数类的决策边界过于狭窄，甚至“记住”这些重复样本，造成过拟合。更严重的是，如果少数类中本身存在噪声点，过采样会放大这些噪声的影响。

#### 代码示例

```python
import numpy as np
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report
from imblearn.over_sampling import RandomOverSampler

# 生成不平衡数据（1000个样本，少数类占5%）
X, y = make_classification(
    n_samples=1000, n_features=20, n_informative=15,
    n_redundant=5, weights=[0.95], random_state=42
)

# 拆分数据集
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

print(f"原始训练集分布：多数类 {sum(y_train==0)}，少数类 {sum(y_train==1)}")

# 应用随机过采样
ros = RandomOverSampler(random_state=42)
X_train_resampled, y_train_resampled = ros.fit_resample(X_train, y_train)

print(f"过采样后训练集分布：多数类 {sum(y_train_resampled==0)}，少数类 {sum(y_train_resampled==1)}")

# 训练对比
model_raw = RandomForestClassifier(random_state=42)
model_raw.fit(X_train, y_train)

model_ros = RandomForestClassifier(random_state=42)
model_ros.fit(X_train_resampled, y_train_resampled)

# 评估
print("\n原始模型评估：")
print(classification_report(y_test, model_raw.predict(X_test)))

print("\n过采样模型评估：")
print(classification_report(y_test, model_ros.predict(X_test)))
```

### SMOTE：合成少数类样本

#### 概念说明

**SMOTE（Synthetic Minority Oversampling Technique）** 是过采样的进阶版本，它不再简单复制样本，而是通过插值生成**合成样本**。具体来说，SMOTE在少数类样本的特征空间中，找到每个样本的k个近邻，然后在样本与近邻的连线上随机选择一点，作为新样本。

#### 原理解析

SMOTE的核心思想是“在特征空间中填充空白”。假设少数类样本在二维特征空间中呈现稀疏分布，SMOTE会在这些稀疏点之间生成新点，从而**扩展少数类的决策区域**。这比随机过采样更鲁棒，因为：

1. **引入多样性**：合成样本不是原始样本的副本，而是结合了多个近邻的特征，使得模型能学到更泛化的模式。
2. **缓解过拟合**：由于样本是“虚拟”生成的，模型不会简单地记住数据点。
3. **保持数据分布**：合成样本位于原始少数类样本的凸包内，不会偏离真实分布太远。

**SMOTE的局限**：它假设少数类样本的特征空间是连续的且近邻关系有意义。如果特征维度很高（如文本、图像），或少数类样本极度稀疏（如只有2-3个样本），SMOTE的效果会下降。此外，SMOTE对噪声敏感——如果少数类中存在离群点，合成样本可能将噪声区域放大。

#### 代码示例

```python
from imblearn.over_sampling import SMOTE
from imblearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

# 注意：SMOTE需要先对特征进行标准化，因为它是基于距离的算法
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('smote', SMOTE(random_state=42, k_neighbors=5)),
    ('classifier', RandomForestClassifier(random_state=42))
])

# 使用Pipeline避免数据泄露
pipeline.fit(X_train, y_train)

print("\nSMOTE模型评估：")
print(classification_report(y_test, pipeline.predict(X_test)))

# 对比不同k值的影响
for k in [3, 5, 10]:
    smote = SMOTE(random_state=42, k_neighbors=k)
    X_smote, y_smote = smote.fit_resample(X_train, y_train)
    model = RandomForestClassifier(random_state=42)
    model.fit(X_smote, y_smote)
    print(f"\nk={k}时的少数类召回率：{classification_report(y_test, model.predict(X_test), output_dict=True)['1']['recall']:.3f}")
```

### 代价敏感学习：从算法层面调整权重

#### 概念说明

**代价敏感学习（Cost-Sensitive Learning）** 不改变数据分布，而是通过修改损失函数或训练过程，让模型对少数类的错误分类付出更高代价。在分类任务中，这通常表现为**类别权重（Class Weight）**——对少数类样本的误分类惩罚更大。

#### 原理解析

代价敏感学习的数学基础是：**最小化加权损失函数**。以二分类交叉熵为例，标准损失函数为：
```
Loss = -[y * log(p) + (1-y) * log(1-p)]
```

加入类别权重后变为：
```
Loss = -[w_pos * y * log(p) + w_neg * (1-y) * log(1-p)]
```
其中 `w_pos` 和 `w_neg` 分别代表正负类的权重。通常设置 `w_pos = N_neg / N_pos`，即让少数类的权重等于多数类样本数除以少数类样本数，从而平衡两类在损失中的贡献。

**代价敏感学习的优势**：
1. **无需修改数据**：避免了过采样可能引入的噪声和过拟合问题。
2. **直接优化目标**：如果业务上更关注少数类的召回率（如欺诈检测），可以针对性地提高其权重。
3. **与模型无关**：大多数机器学习框架（如sklearn、XGBoost、PyTorch）都支持类别权重设置。

**局限性**：当数据极度不平衡（如1:1000）时，单纯调整权重可能效果有限，因为模型仍然缺乏足够的信息来学习少数类的特征边界。此时通常需要结合过采样方法。

#### 代码示例

```python
# 方法1：sklearn内置的class_weight
model_weighted = RandomForestClassifier(
    random_state=42,
    class_weight='balanced'  # 自动根据样本比例设置权重
)
model_weighted.fit(X_train, y_train)

print("\n代价敏感模型评估（sklearn自动权重）：")
print(classification_report(y_test, model_weighted.predict(X_test)))

# 方法2：手动设置权重（更灵活）
# 假设多数类:少数类 = 19:1，则少数类权重设为19
manual_weights = {0: 1.0, 1: 19.0}
model_manual = RandomForestClassifier(
    random_state=42,
    class_weight=manual_weights
)
model_manual.fit(X_train, y_train)

print("\n代价敏感模型评估（手动权重）：")
print(classification_report(y_test, model_manual.predict(X_test)))

# 方法3：在PyTorch中实现
import torch
import torch.nn as nn

# 计算权重
n_neg = (y_train == 0).sum()
n_pos = (y_train == 1).sum()
pos_weight = n_neg / n_pos  # 少数类权重

# 使用带权重的交叉熵损失
criterion = nn.BCEWithLogitsLoss(pos_weight=torch.tensor([pos_weight]))

# 训练循环中正常使用即可
# 注意：pos_weight参数让正类（少数类）的损失乘以该系数
```

## 知识依赖说明

理解本文需要以下前置知识：

1. **分类模型基础**（阶段1）：了解逻辑回归、决策树、随机森林等分类器的基本原理，知道模型如何输出预测概率。
2. **模型评估**（阶段2「机器学习核心」）：掌握混淆矩阵（TP、FP、TN、FN）、精确率（Precision）、召回率（Recall）、F1分数、ROC-AUC等评估指标。这是判断不平衡问题是否被改善的基础。
3. **过拟合与欠拟合**（阶段1）：理解过采样本可能加剧过拟合，而欠采样可能导致信息丢失。
4. **特征缩放**（阶段1）：SMOTE等基于距离的方法要求特征在同一尺度上，需要掌握标准化/归一化。

## 实践建议

1. **先评估，再处理**：不要盲目使用过采样或权重调整。先查看训练集的类别分布，并用混淆矩阵评估原始模型的表现。**如果少数类召回率极低（如<0.1），说明不平衡问题严重，需要干预。**

2. **从代价敏感学习开始**：对于大多数场景，设置`class_weight='balanced'`是最简单且有效的一步。它能快速改善模型对少数类的关注度，且不会引入数据修改的副作用。如果效果不理想，再考虑SMOTE。

3. **SMOTE需要配合交叉验证**：使用SMOTE时务必将其放在Pipeline中，并在交叉验证的每一折内独立执行。如果先对整个训练集做SMOTE再划分验证集，会造成**数据泄露**——验证集中会包含合成样本的“近亲”，导致评估结果虚高。

4. **关注业务代价**：不平衡问题的核心是**误分类代价不对称**。在欺诈检测中，漏掉一笔欺诈（FN）的代价可能是误报一笔正常交易（FP）的100倍。根据业务代价调整权重，而不是机械地使用样本比例。

5. **结合阈值调整**：模型输出的是概率，默认阈值0.5可能不适合不平衡数据。在训练后，使用验证集找到使F1分数最大化的最优阈值。这通常比调整数据或权重更直接有效。

6. **尝试集成方法**：对于极端不平衡（如1:1000），单模型可能力不从心。可以尝试**Bagging + 欠采样**（如EasyEnsemble）或**Boosting + 代价敏感**（如XGBoost的`scale_pos_weight`参数），这些方法通过多个弱分类器的组合来提升少数类的稳定性。

## 延伸阅读

1. **《Learning from Imbalanced Data Sets》**（He & Garcia, 2009）——该领域的经典综述，系统梳理了过采样、欠采样、代价敏感、集成方法等各类技术，是理解不平衡问题的必读文献。

2. **imbalanced-learn文档**（https://imbalanced-learn.org/）——Python中处理不平衡数据的标准库，包含SMOTE的各种变体（如Borderline-SMOTE、ADASYN）、欠采样方法（如Tomek Links、Edited Nearest Neighbors）以及组合Pipeline的详细教程。

3. **《SMOTE: Synthetic Minority Over-sampling Technique》**（Chawla et al., 2002）——SMOTE的原始论文，深入理解合成样本的生成机制和数学原理，对于理解后续变体（如SMOTE-NC处理类别特征、SVMSMOTE支持向量边界）很有帮助。
