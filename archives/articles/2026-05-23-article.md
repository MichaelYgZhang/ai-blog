# 迁移学习与预训练模型使用

## 核心结论

**迁移学习通过将在大规模数据集上预训练的模型知识迁移到目标任务，能够显著降低深度学习模型对标注数据的需求，并将训练效率提升10倍以上，是当前解决小样本学习问题的最有效技术手段。**

## 为什么重要

在深度学习实践中，我们经常面临一个现实困境：工业级应用往往无法获得像ImageNet那样百万级别的标注数据。对于大多数中小规模项目，收集和标注数千甚至数万张图片的成本已经难以承受。**迁移学习的出现彻底改变了这一局面**——它允许我们站在巨人的肩膀上，利用已经在海量数据上训练好的模型，通过少量调整即可解决新问题。

从AI工程师的成长路径来看，迁移学习是一个关键的分水岭。在阶段3「深度学习精通」中，你已经掌握了CNN的基础原理，能够从零开始构建和训练卷积神经网络。但真正让模型从实验室走向工业应用的，正是迁移学习能力。**掌握迁移学习意味着你不再需要每次都为新任务重新训练整个网络**，而是可以高效地复用已有知识，这直接决定了你能否在有限资源下交付高质量的AI解决方案。

迁移学习解决的核心矛盾是**数据稀缺与模型性能之间的矛盾**。它通过领域适配技术，将源域（如ImageNet）中学到的通用特征（边缘、纹理、形状等）迁移到目标域（如医学影像、卫星图像）中，使得即便目标域只有几百张图片，也能训练出具有实用价值的深度学习模型。

## 核心内容

### 特征提取：冻结预训练模型作为特征提取器

#### 概念说明

特征提取是最基础的迁移学习策略。**核心思想是将预训练模型视为一个固定的特征提取器**，移除其顶部的分类层，然后使用它从新数据中提取特征向量，最后在这些特征上训练一个新的分类器。

#### 原理解析

在CNN的层级结构中，底层网络（靠近输入的层）学习的是通用特征，如边缘、颜色斑点、纹理等；中层网络学习的是更具组合性的特征，如物体部件、图案；顶层网络则学习与特定任务高度相关的语义特征，如人脸、车轮等。**特征提取策略利用了底层特征的通用性**——这些特征在不同视觉任务中几乎都可以复用。

假设我们在ImageNet上预训练了一个ResNet-50模型。当我们将其应用于新的分类任务时，可以冻结所有卷积层的参数，只训练最后新添加的全连接层。这样做的好处是：
- **计算量大幅降低**：只需要反向传播更新少量参数
- **避免过拟合**：冻结的卷积层参数已经过充分训练，不会因为小数据集而退化
- **通用性保证**：底层特征在不同数据集间具有高可迁移性

#### 代码示例

```python
import torch
import torch.nn as nn
import torchvision.models as models
from torch.utils.data import DataLoader

# 加载预训练ResNet-50，不使用预训练权重时设置为pretrained=False
model = models.resnet50(pretrained=True)

# 冻结所有参数
for param in model.parameters():
    param.requires_grad = False

# 替换最后的全连接层，适应新的分类任务（假设有10个类别）
num_features = model.fc.in_features
model.fc = nn.Linear(num_features, 10)

# 定义损失函数和优化器（只优化新添加的全连接层）
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.SGD(model.fc.parameters(), lr=0.001)

# 训练循环
def train_one_epoch(dataloader, model, criterion, optimizer):
    model.train()
    running_loss = 0.0
    for inputs, labels in dataloader:
        optimizer.zero_grad()
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()
        running_loss += loss.item()
    return running_loss / len(dataloader)

# 注意：实际使用时需要准备自己的DataLoader
```

### 微调策略：精细调整预训练模型

#### 概念说明

微调（Fine-tuning）是比特征提取更灵活的策略。**它不仅训练新添加的分类层，还以较小的学习率解冻并更新部分或全部预训练层的参数**，使模型更好地适应目标域的数据分布。

#### 原理解析

当目标域与源域差异较大时（例如从自然图像迁移到医学X光图像），仅使用固定特征提取器可能不够充分。微调允许模型对底层特征进行适当调整，使其更好地捕捉目标域的特有模式。

微调的关键在于**控制调整的幅度和范围**。通常的策略包括：
1. **渐进式解冻**：先训练新分类层几个epoch，然后从顶层开始逐层解冻卷积层
2. **差异化学习率**：对底层使用极小学习率（如1e-5），对顶层使用稍大学习率（如1e-4）
3. **选择性微调**：只微调最后几个残差块或卷积层，保持底层完全冻结

**微调的风险在于过拟合**，特别是当目标数据集非常小时。因此，通常建议在特征提取效果不佳时再考虑微调，且微调时需要使用比正常训练小10倍以上的学习率。

#### 代码示例

```python
import torch.nn as nn
import torch.optim as optim
from torchvision import models

# 加载预训练模型
model = models.resnet50(pretrained=True)

# 替换分类层
num_features = model.fc.in_features
model.fc = nn.Linear(num_features, 10)

# 策略一：分层设置学习率
# 冻结前几层，只微调最后两个残差块和分类层
def set_parameter_requires_grad(model, feature_extracting=True):
    if feature_extracting:
        for param in model.parameters():
            param.requires_grad = False

set_parameter_requires_grad(model, feature_extracting=True)

# 解冻最后两个残差块（layer4和layer3）
for name, child in model.named_children():
    if name in ['layer4', 'layer3']:
        for param in child.parameters():
            param.requires_grad = True

# 解冻分类层
for param in model.fc.parameters():
    param.requires_grad = True

# 策略二：差异化学习率
# 将参数分组，对不同组使用不同学习率
params_to_update = []
params_to_finetune = []

for name, param in model.named_parameters():
    if param.requires_grad:
        if 'fc' in name or 'layer4' in name:
            params_to_finetune.append(param)
        else:
            params_to_update.append(param)

optimizer = optim.SGD([
    {'params': params_to_update, 'lr': 1e-5},
    {'params': params_to_finetune, 'lr': 1e-4}
], momentum=0.9)

# 训练时使用更小的学习率
scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, mode='min', factor=0.1, patience=3)
```

### 领域适配：跨越数据分布差异

#### 概念说明

领域适配（Domain Adaptation）是迁移学习的高级形式，**专门处理源域和目标域数据分布不一致的问题**。当两个领域的数据特征存在系统性差异时（如合成图像vs真实图像），直接迁移会导致性能下降。

#### 原理解析

领域适配的核心挑战在于**减小源域和目标域之间的分布差异**。常用的方法包括：
1. **对抗性领域适配**：引入一个领域判别器，让特征提取器学习到领域不变的特征表示
2. **最大均值差异（MMD）**：在损失函数中加入MMD项，惩罚两个领域特征分布之间的差异
3. **自监督预训练**：在目标域上进行自监督学习（如旋转预测、对比学习），作为迁移的桥梁

其中，对抗性领域适配（如Domain-Adversarial Neural Network, DANN）是最具代表性的方法。其核心思想是：特征提取器不仅要完成分类任务，还要让领域判别器无法区分特征来自源域还是目标域，从而迫使模型学习到领域无关的通用特征。

#### 代码示例

```python
import torch.nn as nn
import torch.nn.functional as F
from torch.autograd import Function

# 梯度反转层（Gradient Reversal Layer）
class GradientReversalLayer(Function):
    @staticmethod
    def forward(ctx, x, lambda_=1.0):
        ctx.lambda_ = lambda_
        return x.view_as(x)
    
    @staticmethod
    def backward(ctx, grad_output):
        return grad_output.neg() * ctx.lambda_, None

# 领域对抗网络
class DomainAdversarialNetwork(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        # 特征提取器（使用预训练模型的前几层）
        resnet = models.resnet18(pretrained=True)
        self.feature_extractor = nn.Sequential(*list(resnet.children())[:-2])
        
        # 分类器
        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool2d((1, 1)),
            nn.Flatten(),
            nn.Linear(512, num_classes)
        )
        
        # 领域判别器
        self.domain_discriminator = nn.Sequential(
            nn.Linear(512, 256),
            nn.ReLU(),
            nn.Dropout(0.5),
            nn.Linear(256, 2)  # 二分类：源域 vs 目标域
        )
    
    def forward(self, x, lambda_=0.5):
        features = self.feature_extractor(x)
        features_pooled = F.adaptive_avg_pool2d(features, (1, 1)).view(features.size(0), -1)
        
        # 分类输出
        class_output = self.classifier(features)
        
        # 领域输出（应用梯度反转）
        reversed_features = GradientReversalLayer.apply(features_pooled, lambda_)
        domain_output = self.domain_discriminator(reversed_features)
        
        return class_output, domain_output

# 训练时联合优化两个损失
def train_dann(model, source_loader, target_loader, optimizer, epoch):
    model.train()
    lambda_ = 2.0 / (1.0 + torch.exp(torch.tensor(-10.0 * epoch / 100))) - 1.0
    
    for (source_data, source_labels), (target_data, _) in zip(source_loader, target_loader):
        # 组合源域和目标域数据
        combined_data = torch.cat([source_data, target_data], dim=0)
        
        # 领域标签：源域为0，目标域为1
        domain_labels = torch.cat([
            torch.zeros(source_data.size(0), dtype=torch.long),
            torch.ones(target_data.size(0), dtype=torch.long)
        ])
        
        optimizer.zero_grad()
        class_output, domain_output = model(combined_data, lambda_)
        
        # 分类损失（只对源域数据计算）
        class_loss = F.cross_entropy(class_output[:source_data.size(0)], source_labels)
        # 领域损失
        domain_loss = F.cross_entropy(domain_output, domain_labels)
        
        total_loss = class_loss + domain_loss
        total_loss.backward()
        optimizer.step()
```

理解本文需要以下前置知识：

1. **CNN基础**（阶段3「深度学习精通」）：这是理解迁移学习的基石。你需要熟悉卷积层、池化层、全连接层的工作原理，以及ResNet、VGG等经典架构的结构特点。特别是**层级的特征表示**概念，即底层学习通用特征、顶层学习语义特征，这是理解为何能迁移的关键。

2. **PyTorch基础**：包括模型定义、参数管理（`requires_grad`）、优化器配置等基本操作。上述代码示例中的参数冻结、分层学习率设置等操作都需要这些基础。

3. **过拟合与正则化**：理解为什么在小数据集上微调容易过拟合，以及Dropout、权重衰减等正则化技术的作用。

## 实践建议

1. **从特征提取开始，再考虑微调**：对于大多数实际项目，先尝试冻结全部卷积层只训练分类层。如果验证集性能不理想，再逐步解冻顶层进行微调。**这是最稳妥的迁移学习策略**，能有效避免过拟合。

2. **使用分层学习率**：微调时，对底层使用1e-5或更小的学习率，对顶层使用1e-4左右的学习率。**一个实用的经验法则是：越靠近输入的层，学习率越小**。可以尝试将学习率设置为逐层衰减的序列。

3. **数据预处理保持一致**：确保目标域数据的预处理方式与预训练模型训练时一致。例如，ImageNet预训练模型通常要求输入图像归一化到mean=[0.485, 0.456, 0.406]，std=[0.229, 0.224, 0.225]。**预处理不一致会严重损害迁移效果**。

4. **监控领域差异**：当源域和目标域差异较大时（如从自然图像迁移到医学图像），使用t-SNE等可视化工具观察特征分布。如果两个领域的特征在特征空间中明显分离，说明需要更强的领域适配技术。

5. **尝试不同的预训练模型**：不要局限于ResNet系列。对于小数据集，轻量级模型如MobileNet、EfficientNet可能更适合；对于高精度要求，可以考虑更大的模型如ViT（Vision Transformer）。**选择合适的预训练模型是迁移学习成功的关键第一步**。
