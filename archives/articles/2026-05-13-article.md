# PyTorch入门：Tensor与自动求导

## 核心结论

**PyTorch的核心竞争力在于其动态计算图与自动求导机制，这使得深度学习模型的训练过程从手动推导梯度、手写反向传播的繁琐工作中彻底解放出来，让研究者可以专注于模型架构的创新。**

## 为什么重要

在深度学习的实践中，**梯度计算是模型训练的数学基石**。无论是简单的线性回归还是复杂的Transformer，其核心优化算法（如SGD、Adam）都依赖于损失函数对模型参数的梯度。在PyTorch出现之前，研究者需要手动推导每个层的梯度公式，并编写复杂的反向传播代码，这不仅效率低下，而且极易出错，严重阻碍了深度学习模型的快速迭代。

PyTorch通过其核心组件——**Tensor**和**autograd**——优雅地解决了这个痛点。Tensor作为神经网络中数据流动的基本单元，它不仅仅是多维数组，更是一个能够记录所有操作历史的“智能”数据结构。而autograd引擎则利用这些记录自动构建计算图，并在需要时高效地计算所有参数的梯度。

理解Tensor和autograd，**是掌握PyTorch乃至整个深度学习框架的钥匙**。它让你从“如何实现反向传播”的底层细节中抽身，转而聚焦于“如何设计更好的模型”这一更高层次的问题。这不仅是技术上的进步，更是思维方式的转变——从“手动编程”到“声明式定义”，让AI工程师的创造潜力得到最大释放。

## 核心内容

### 一、Tensor：深度学习的基本数据结构

#### 概念说明

**Tensor（张量）** 是PyTorch中最基本的数据结构，可以理解为NumPy中`ndarray`的深度学习升级版。一个Tensor可以是一个标量（0维）、向量（1维）、矩阵（2维）或任意高维数组。它与NumPy数组的核心区别在于：**Tensor可以运行在GPU上加速计算，并且能够自动记录用于梯度计算的操作历史。**

```python
import torch
import numpy as np

# 从Python列表创建
tensor_1d = torch.tensor([1, 2, 3])
tensor_2d = torch.tensor([[1, 2], [3, 4]])

# 从NumPy数组创建
np_array = np.array([1, 2, 3])
tensor_from_np = torch.from_numpy(np_array)

# 创建特殊张量
zeros = torch.zeros(2, 3)      # 全0张量
ones = torch.ones(2, 3)        # 全1张量
rand = torch.rand(2, 3)        # 均匀分布随机张量
randn = torch.randn(2, 3)      # 标准正态分布随机张量
```

#### 原理解析

Tensor在内存中的存储方式与NumPy类似，但增加了几个关键属性：
- **`dtype`**：数据类型，如`torch.float32`、`torch.int64`
- **`device`**：存储设备，`cpu`或`cuda`（GPU）
- **`requires_grad`**：是否需要计算梯度，这是Tensor最关键的深度学习特性
- **`grad`**：存储计算出的梯度值
- **`grad_fn`**：记录创建该Tensor的函数，构成计算图的节点

Tensor的操作遵循**函数式编程**理念，大多数操作都会返回新的Tensor，而不是修改原Tensor。这种不可变性保证了计算图的可追踪性。

#### 代码示例与实践

```python
# Tensor基本运算
a = torch.tensor([1, 2, 3], dtype=torch.float32)
b = torch.tensor([4, 5, 6], dtype=torch.float32)

# 逐元素运算
c = a + b        # 加法
d = a * b        # 逐元素乘法
e = torch.matmul(a, b)  # 矩阵乘法（点积）

# 广播机制（与NumPy一致）
f = torch.tensor([[1, 2], [3, 4]])
g = torch.tensor([10, 20])
h = f + g        # g被广播为[[10, 20], [10, 20]]
print(h)         # 输出: tensor([[11, 22], [13, 24]])

# GPU加速（如果可用）
if torch.cuda.is_available():
    device = torch.device('cuda')
    tensor_gpu = torch.tensor([1, 2, 3]).to(device)
    result = tensor_gpu * 2  # 在GPU上计算
    print(result)            # 输出: tensor([2, 4, 6], device='cuda:0')
```

### 二、autograd：自动梯度计算引擎

#### 概念说明

**autograd**是PyTorch的自动微分系统，它能够自动计算神经网络中所有参数关于损失函数的梯度。其核心思想是：**记录所有Tensor操作，构建一个动态计算图，然后通过链式法则自动计算梯度。**

要启用自动梯度追踪，只需在创建Tensor时设置`requires_grad=True`：

```python
# 创建需要梯度的Tensor
x = torch.tensor([2.0], requires_grad=True)
w = torch.tensor([3.0], requires_grad=True)
b = torch.tensor([1.0], requires_grad=True)

# 构建计算图：y = w * x + b
y = w * x + b

# 计算梯度
y.backward()

# 查看梯度
print(f"dy/dx = {x.grad}")    # 输出: dy/dx = tensor([3.])
print(f"dy/dw = {w.grad}")    # 输出: dy/dw = tensor([2.])
print(f"dy/db = {b.grad}")    # 输出: dy/db = tensor([1.])
```

#### 原理解析

autograd的工作原理基于**动态计算图**：

1. **构建阶段**：当执行`y = w * x + b`时，PyTorch自动记录这是一个加法操作，并创建计算图节点。每个节点记录：
   - 输入Tensor
   - 输出Tensor
   - 操作类型（如加法、乘法）
   - 指向父节点的引用

2. **前向传播**：按照计算图从输入到输出执行计算，同时保存中间结果。

3. **反向传播**：调用`backward()`时，从输出节点开始，**按照计算图的逆向拓扑顺序**，利用链式法则逐层计算梯度。

关键特性：
- **动态图**：计算图在每次前向传播时重新构建，这使得模型结构可以动态变化（如循环、条件分支）
- **梯度累积**：默认情况下，梯度会累积到`.grad`属性中，每次反向传播前需要手动清零
- **非标量梯度**：如果`backward()`的起始Tensor不是标量，需要传入一个同形状的梯度权重

```python
# 动态计算图示例
x = torch.tensor([1.0, 2.0], requires_grad=True)

# 第一次前向传播
y = x ** 2
z = y.sum()
z.backward()
print(f"First backward: {x.grad}")  # tensor([2., 4.])

# 清零梯度（重要！）
x.grad.zero_()

# 第二次前向传播（不同结构）
y = x * 3
z = y.mean()
z.backward()
print(f"Second backward: {x.grad}")  # tensor([1.5, 1.5])
```

### 三、计算图：深度学习模型的骨架

#### 概念说明

**计算图**是一个有向无环图（DAG），其中节点代表Tensor或操作，边代表数据流向。它是深度学习框架的核心抽象，**将复杂的数学计算过程转化为可追踪、可微分的图结构**。

#### 原理解析

计算图包含两种节点：
- **叶子节点**：由用户直接创建的Tensor（如模型参数、输入数据）
- **中间节点**：由运算产生的Tensor

每个节点维护：
- `data`：存储实际数值
- `grad`：存储梯度值
- `grad_fn`：指向创建该节点的函数（如`AddBackward0`）
- `next_functions`：指向输入节点的`grad_fn`

```python
# 可视化计算图结构
x = torch.tensor(2.0, requires_grad=True)
w = torch.tensor(3.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)

# 构建计算图
a = w * x      # 乘法节点
y = a + b      # 加法节点

# 查看计算图信息
print(f"y.grad_fn: {y.grad_fn}")          # <AddBackward0 object>
print(f"a.grad_fn: {a.grad_fn}")          # <MulBackward0 object>
print(f"x.grad_fn: {x.grad_fn}")          # None（叶子节点）
print(f"y.grad_fn.next_functions: {y.grad_fn.next_functions}")  # 显示输入节点
```

**反向传播过程**：
1. 从`y.grad_fn`（加法）开始，计算`dy/da`和`dy/db`
2. 进入`a.grad_fn`（乘法），计算`da/dw`和`da/dx`
3. 利用链式法则组合得到最终的梯度

```python
# 手动模拟反向传播
# dy/da = 1, dy/db = 1
# da/dw = x = 2, da/dx = w = 3
# dy/dw = dy/da * da/dw = 1 * 2 = 2
# dy/dx = dy/da * da/dx = 1 * 3 = 3
# dy/db = 1

y.backward()
print(f"dy/dx = {x.grad}")    # tensor(3.)
print(f"dy/dw = {w.grad}")    # tensor(2.)
print(f"dy/db = {b.grad}")    # tensor(1.)
```

### 四、实战：线性回归模型

将上述知识整合到一个完整的线性回归示例中：

```python
import torch
import torch.nn as nn

# 1. 生成数据
torch.manual_seed(42)
X = torch.randn(100, 1) * 10
true_w = torch.tensor([[2.0]])
true_b = torch.tensor([5.0])
y = X @ true_w + true_b + torch.randn(100, 1) * 2  # 添加噪声

# 2. 定义模型参数（需要梯度）
w = torch.randn(1, 1, requires_grad=True)
b = torch.zeros(1, requires_grad=True)

# 3. 训练循环
learning_rate = 0.01
n_epochs = 1000

for epoch in range(n_epochs):
    # 前向传播
    y_pred = X @ w + b
    loss = ((y_pred - y) ** 2).mean()  # MSE损失
    
    # 反向传播
    loss.backward()
    
    # 梯度下降更新参数
    with torch.no_grad():  # 临时关闭梯度追踪
        w -= learning_rate * w.grad
        b -= learning_rate * b.grad
        
        # 清零梯度
        w.grad.zero_()
        b.grad.zero_()
    
    if epoch % 100 == 0:
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")

print(f"Learned w: {w.item():.4f}, True w: {true_w.item():.4f}")
print(f"Learned b: {b.item():.4f}, True b: {true_b.item():.4f}")
```

## 知识依赖说明

理解本文内容需要以下前置知识：

1. **Python基础**：变量、函数、面向对象编程（OOP）概念
2. **NumPy基础**：多维数组操作、广播机制、矩阵运算。Tensor的API设计与NumPy高度一致，熟悉NumPy能大幅降低学习曲线
3. **微积分基础**：导数、偏导数、链式法则。虽然autograd自动完成了梯度计算，但理解其背后的数学原理有助于调试和优化模型
4. **线性代数基础**：矩阵乘法、向量运算。神经网络本质上是多层矩阵运算的复合

建议在学习本文前，先复习阶段1中的相关内容，特别是NumPy的数组操作和广播机制。

## 实践建议

1. **从简单开始**：先使用`requires_grad=True`构建一个简单的`y = w*x + b`模型，手动验证梯度计算的正确性，理解计算图的构建过程

2. **养成清零梯度的习惯**：每次`backward()`后，务必调用`optimizer.zero_grad()`或手动清零`.grad`，否则梯度会累积导致训练失败

3. **善用`torch.no_grad()`**：在模型评估、参数更新等不需要梯度的场景下，使用`with torch.no_grad():`包装代码，可以节省内存并加速计算

4. **可视化计算图**：使用`torchviz.make_dot()`或TensorBoard的图形功能，可视化你的计算图，这对于调试复杂模型非常有帮助

5. **从手动到自动的过渡**：先用autograd实现一个简单网络，然后对比使用`nn.Module`和`nn.Linear`的版本，理解框架如何进一步封装和简化开发流程
