## 3. 核心内容

### 3.1 论文精读：超越表面，洞察设计逻辑
论文精读不是简单的翻译，而是带着实现和质疑的眼光进行结构化解析。

**概念说明**：精读的目标是提取出模型的**完整计算图**、**所有超参数**以及**关键的训练细节**。你需要区分哪些是核心创新（如Scaled Dot-Product Attention），哪些是工程性选择（如学习率预热策略）。

**原理解析**：
1.  **结构化梳理**：将论文分解为几个模块：模型架构（Encoder-Decoder, Attention, FFN）、训练目标与细节、实验结果。为每个模块制作一张“事实表”，记录所有明确的定义、公式和参数值。
2.  **追问细节**：论文往往省略“常识”或“默认”设置。例如，论文提到使用“残差连接和LayerNorm”，但顺序是什么？是 `Pre-Norm` (Norm -> Sublayer -> Residual) 还是 `Post-Norm` (Sublayer -> Residual -> Norm)？原始论文实际采用的是 `Post-Norm`，但后来 `Pre-Norm` 被证明更稳定而被广泛采用。
3.  **理解动机**：为什么是“Scaled” Dot-Product？论文指出，当维度 `dk` 较大时，点积结果方差变大，将softmax推入梯度极小的区域，因此用 `sqrt(dk)` 进行缩放以稳定梯度。

**实践指导**：
- 使用双栏笔记法：一栏摘录原文关键句/公式，另一栏记录自己的理解、疑问和实现注解。
- 重点标注所有出现数字的地方（维度、层数、头数、dropout率、学习率、batch size等），这些将是复现的蓝图。
- 绘制出完整的数据流图，从输入嵌入开始，跟踪一个数据样本如何经过编码器、解码器，最终生成输出。

### 3.2 代码复现：从蓝图到可运行的模型
复现不是抄袭现有实现，而是依据论文描述，从零开始构建。

**概念说明**：代码复现分为**模型架构搭建**和**训练循环实现**两部分。目标是得到一个与论文描述在数学上等价的模型，并能在一个小规模数据集上成功过拟合。

**原理解析与代码示例**：
我们将使用PyTorch进行关键组件的实现。

**1. 缩放点积注意力（Scaled Dot-Product Attention）**：
这是Transformer最核心的操作。其公式为：`Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V`

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class ScaledDotProductAttention(nn.Module):
    """缩放点积注意力"""
    def __init__(self, dropout=0.1):
        super().__init__()
        self.dropout = nn.Dropout(dropout)

    def forward(self, query, key, value, mask=None):
        # query, key, value: [batch_size, num_heads, seq_len, d_k]
        d_k = query.size(-1)
        # 计算 QK^T / sqrt(d_k)
        scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)

        # 应用掩码（如解码器的自回归掩码或padding掩码）
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)

        # 计算注意力权重
        attn_weights = F.softmax(scores, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # 加权求和
        output = torch.matmul(attn_weights, value) # [batch_size, num_heads, seq_len, d_v]
        return output, attn_weights
```

**2. 多头注意力（Multi-Head Attention）**：
将输入线性投影到h个不同的子空间，分别进行注意力计算后拼接，再投影回原维度。

```python
class MultiHeadAttention(nn.Module):
    """多头注意力"""
    def __init__(self, d_model, num_heads, dropout=0.1):
        super().__init__()
        assert d_model % num_heads == 0, "d_model must be divisible by num_heads"
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        # 线性投影层
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

        self.attention = ScaledDotProductAttention(dropout)
        self.dropout = nn.Dropout(dropout)
        self.layer_norm = nn.LayerNorm(d_model)

    def split_heads(self, x):
        """将最后一维 (d_model) 分割为 (num_heads, d_k)"""
        batch_size, seq_len, _ = x.size()
        return x.view(batch_size, seq_len, self.num_heads, self.d_k).transpose(1, 2)

    def combine_heads(self, x):
        """合并多头"""
        batch_size, _, seq_len, _ = x.size()
        return x.transpose(1, 2).contiguous().view(batch_size, seq_len, self.d_model)

    def forward(self, query, key, value, mask=None):
        # 残差连接
        residual = query

        # 1. 线性投影并分割多头
        query = self.split_heads(self.W_q(query))
        key = self.split_heads(self.W_k(key))
        value = self.split_heads(self.W_v(value))

        # 2. 应用缩放点积注意力
        attn_output, attn_weights = self.attention(query, key, value, mask)
        attn_output = self.combine_heads(attn_output)

        # 3. 输出投影
        output = self.W_o(attn_output)
        output = self.dropout(output)

        # 4. 残差连接与层归一化 (Post-Norm，遵循原论文)
        output = self.layer_norm(output + residual)
        return output, attn_weights
```

**3. 前馈网络（Position-wise Feed-Forward Network）**：
这是一个应用于每个位置的两层MLP，中间有ReLU激活和维度扩展。

```python
class PositionwiseFFN(nn.Module):
    """位置式前馈网络"""
    def __init__(self, d_model, d_ff, dropout=0.1):
        super().__init__()
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)
        self.dropout = nn.Dropout(dropout)
        self.layer_norm = nn.LayerNorm(d_model)
        self.activation = nn.ReLU()

    def forward(self, x):
        residual = x
        x = self.linear1(x)
        x = self.activation(x)
        x = self.dropout(x)
        x = self.linear2(x)
        x = self.dropout(x)
        x = self.layer_norm(x + residual) # Post-Norm
        return x
```

**实践指导**：
- **模块化开发**：像上面一样，先实现最小可验证单元（如Attention），用简单的输入测试其形状和基本功能。
- **过拟合一个小数据集**：使用一个极小的合成数据集（如复制任务），目标是让训练损失快速下降到接近0。这是验证模型**前向传播和反向传播**正确性的最快方法。
- **严格对照论文**：在实现编码器层、解码器层、嵌入、位置编码等每个部分时，不断回头核对论文中的维度、公式和参数设置。

### 3.3 消融实验：验证“注意力就是一切”的核心主张
消融实验是科学验证的灵魂，用于量化每个组件对最终性能的贡献。

**概念说明**：消融实验通过系统地移除或修改模型的某个组件，观察性能变化，从而理解该组件的重要性。对于Transformer，核心主张是“自注意力机制足以处理序列建模”，因此我们需要挑战或验证这一点。

**原理解析**：
论文本身已经包含了一些消融实验，例如：
- 比较不同注意力头数（`h`）的影响。
- 比较注意力函数（加性注意力 vs. 点积注意力）。
- 分析注意力权重在不同层、不同头上的分布。

在复现中，我们可以设计自己的消融实验：

**实践指导与实验设计**：
1.  **替换注意力机制**：将多头自注意力替换为RNN（如LSTM）或CNN层，保持其他部分（FFN、层数、参数量）大致不变。在同一个翻译任务上比较BLEU分数。**预期结果**：纯RNN/CNN的性能应显著低于注意力机制，尤其是在长序列任务上。
2.  **移除位置编码**：将正弦位置编码设为零或移除。**预期结果**：模型性能会严重下降，因为自注意力本身是位置无关的。
3.  **分析多头的重要性**：固定总计算量（`d_model`不变），比较不同头数（如 `h=1, 4, 8, 16`）的性能。可视化不同头的注意力图，观察它们是否学习了不同的关注模式（如语法、指代、语义）。
4.  **缩放因子的验证**：尝试移除缩放因子 `sqrt(d_k)`，观察训练是否变得更不稳定、收敛更慢。

**代码示例（简单的消融实验框架）**：
```python
def run_ablation_experiment(model_variant, dataloader, criterion, optimizer, num_epochs=5):
    """
    运行一个消融实验。
    model_variant: 不同的模型配置（如无位置编码、单头注意力等）
    """
    losses = []
    for epoch in range(num_epochs):
        model_variant.train()
        total_loss = 0
        for batch in dataloader:
            src, tgt = batch
            optimizer.zero_grad()
            output = model_variant(src, tgt[:, :-1]) # 教师强制
            loss = criterion(output.contiguous().view(-1, output.size(-1)),
                             tgt[:, 1:].contiguous().view(-1))
            loss.backward()
            optimizer.step()
            total_loss += loss.item()
        avg_loss = total_loss / len(dataloader)
        losses.append(avg_loss)
        print(f"Variant: {model_variant.name}, Epoch {epoch}, Loss: {avg_loss:.4f}")
    return losses
```

## 5. 实践建议
1.  **从“最小复现”开始**：不要一开始就追求在WMT数据集上完全复现SOTA结果。首先在**微型合成任务**（如将“abc”翻译为“ABC”）上实现模型并成功过拟合，确保模型基础逻辑正确。
2.  **善用开源实现作为“参考答案”**：在你自己努力实现并调试后，再去参考如**Harvard NLP的Annotated Transformer**或**PyTorch官方教程**。对比你们的实现差异，思考为什么他们那么写，这能学到大量工程技巧。
3.  **可视化，可视化，再可视化**：在训练过程中和训练后，可视化注意力权重矩阵、训练损失曲线、梯度分布。这能给你关于模型内部运作最直观的反馈，也是发现问题的利器。
4.  **系统记录实验日志**：使用TensorBoard、Weights & Biases或简单的文本文件，记录每一次运行（基线、消融实验）的所有超参数、最终指标和关键观察。可重复性和可追溯性是科研的基石。
5.  **尝试提出自己的“微创新”**：在完全复现后，可以尝试一个小的改进点，例如将 `Post-Norm` 改为 `Pre-Norm` 并观察训练稳定性，或尝试不同的位置编码（如可学习的位置编码），并设计实验验证其效果。这能让你从复现者向创新者迈出第一步。
