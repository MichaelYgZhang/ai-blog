# LangChain构建AI应用链

## 核心结论
**LangChain的核心价值在于通过“链式组合”将大语言模型（LLM）从一个强大的对话接口，转变为可编程、可集成、具备记忆与逻辑推理能力的应用开发框架，是构建复杂AI应用的事实标准。**

## 为什么重要

在AI工程师的成长路径中，掌握Prompt工程是解锁大模型潜力的第一步，它让我们学会了如何与模型“对话”。然而，当我们需要构建一个真正可用的、复杂的AI应用（如智能客服、数据分析助手、个性化内容生成系统）时，仅靠单次Prompt交互是远远不够的。我们面临三大核心挑战：**如何串联多个步骤形成工作流？如何让应用记住对话历史？如何让模型调用外部工具（如数据库、API、计算器）来获取实时信息或执行动作？**

这正是LangChain所要解决的核心问题。它位于“大模型应用开发”阶段，是连接底层大模型能力与上层业务逻辑的关键桥梁。学习LangChain，意味着AI工程师从“模型使用者”进阶为“应用架构师”，能够系统性地设计、编排和部署基于大模型的智能体（Agent）或应用链（Chain），从而解决真实世界中的复杂任务。它抽象并封装了构建此类应用的通用模式，极大地提升了开发效率与系统的可维护性。

## 核心内容

### 1. Chain组合：从单次调用到工作流编排

**概念说明**
Chain是LangChain中最核心的抽象。它将一个或多个组件（如LLM、Prompt模板、工具、其他Chain）按特定顺序链接起来，形成一个可执行的工作流。一个简单的Chain可能只是“模板填充 -> 调用LLM”，而复杂的Chain可以实现多步推理、条件分支和循环。

**原理解析**
LangChain提供了多种Chain类型：
- **`LLMChain`**: 最基础的链，组合了`PromptTemplate`和`LLM`。
- **`SequentialChain`**: 按顺序执行多个子链，前一个链的输出可作为后一个链的输入。
- **`RouterChain`**: 根据输入内容，决定将其传递给哪个下游链处理，实现动态路由。
- **`TransformChain`**: 允许在链中插入自定义的数据转换函数。

其底层思想是**将应用逻辑分解为可复用的模块，并通过清晰的接口（输入/输出）进行组合**，这符合软件工程的高内聚、低耦合原则。

**代码示例**
```python
from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.chains import LLMChain, SequentialChain

# 1. 定义组件
llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.7)

# 链1：生成公司名称
prompt_1 = ChatPromptTemplate.from_template(
    "我的产品是{product}，请为它起三个朗朗上口的公司名称。"
)
chain_1 = LLMChain(llm=llm, prompt=prompt_1, output_key="company_names")

# 链2：为选定的名称生成口号
prompt_2 = ChatPromptTemplate.from_template(
    "为这个公司名称：'{company_name}' 构思一句品牌口号。"
)
chain_2 = LLMChain(llm=llm, prompt=prompt_2, output_key="slogan")

# 2. 组合成顺序链
overall_chain = SequentialChain(
    chains=[chain_1, chain_2],
    input_variables=["product"],
    # 第一个链的输出变量`company_names`会作为上下文传递给第二个链。
    # 但第二个链需要的是`company_name`，这里需要适配。更复杂的场景可使用`TransformChain`或自定义逻辑。
    output_variables=["company_names", "slogan"],
    verbose=True # 打印执行日志
)

# 3. 执行
result = overall_chain.invoke({"product": "环保可降解手机壳"})
print(f"公司名称建议：{result['company_names']}")
print(f"品牌口号：{result['slogan']}")
```

### 2. Memory管理：让AI拥有“记忆”

**概念说明**
Memory模块使Chain或Agent能够记住和引用历史交互信息。这对于实现多轮对话、个性化体验至关重要。没有Memory，每次对话都是独立的，AI无法引用你之前说过的话。

**原理解析**
LangChain提供了多种Memory类型，核心是将历史对话以某种结构（如列表、摘要、向量）存储，并在构建新Prompt时将其作为上下文注入。
- **`ConversationBufferMemory`**: 最简单，将完整的对话历史存储在缓冲区。
- **`ConversationBufferWindowMemory`**: 只保留最近K轮对话，防止上下文过长。
- **`ConversationSummaryMemory`**: 调用LLM对历史对话进行摘要，只将摘要注入上下文，节省Token。
- **`ConversationEntityMemory`**: 识别并记忆对话中提到的实体（如人、地点、事件）及其属性，实现更结构化的记忆。

**实践指导**
```python
from langchain.memory import ConversationBufferMemory
from langchain.chains import ConversationChain

# 创建带记忆的对话链
memory = ConversationBufferMemory(return_messages=True) # return_messages=True 返回消息对象列表
conversation = ConversationChain(
    llm=llm,
    memory=memory,
    verbose=True
)

# 第一轮对话
response1 = conversation.invoke({"input": "你好，我叫张三。"})
print(response1['response']) # 模型回复，如“你好张三，很高兴认识你！”

# 第二轮对话，模型记得我的名字
response2 = conversation.invoke({"input": "你还记得我叫什么吗？"})
print(response2['response']) # 模型应能回答“你叫张三。”

# 查看当前记忆内容
print(memory.load_memory_variables({}))
```

### 3. 工具集成：突破模型的“知识”与“能力”边界

**概念说明**
工具（Tool）是LangChain中赋予LLM执行具体操作能力的接口，例如搜索网络、查询数据库、运行代码、调用API。通过工具，LLM可以获取训练数据截止日期之后的信息，或执行其本身无法完成的计算和操作。

**原理解析**
工具集成的核心模式是**智能体（Agent）**。Agent是一个特殊的Chain，它由LLM驱动，具备“思考-行动-观察”的循环能力。其工作流程为：
1. **思考**：LLM根据用户输入和当前状态，决定下一步是直接回答，还是调用某个工具。
2. **行动**：如果决定调用工具，则生成工具调用指令（包括工具名和参数）。
3. **观察**：执行工具，获取结果（文本或数据）。
4. **循环**：将工具结果作为新的观察反馈给LLM，LLM继续思考，直到得出最终答案。

**代码示例**
```python
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain.tools import Tool
from langchain_community.utilities import SerpAPIWrapper
from langchain import hub

# 1. 定义工具
# 工具1：网络搜索
search = SerpAPIWrapper()
search_tool = Tool(
    name="Search",
    func=search.run,
    description="当需要回答关于实时信息或最新事件的问题时使用此工具。"
)

# 工具2：自定义计算器（示例）
def custom_calculator(expression: str) -> str:
    """计算一个数学表达式。例如，输入 '2 + 2' 返回 '4'。"""
    try:
        # 警告：实际应用中应使用更安全的评估方法，如`ast.literal_eval`或专用库
        result = eval(expression)
        return str(result)
    except Exception as e:
        return f"计算错误：{e}"

calc_tool = Tool(
    name="Calculator",
    func=custom_calculator,
    description="用于计算数学表达式。输入应是一个有效的数学表达式字符串，如 '2 + 2' 或 'sqrt(16)'。"
)

tools = [search_tool, calc_tool]

# 2. 创建Agent
# 拉取一个预设的Prompt模板，其中包含指导Agent使用工具的指令
prompt = hub.pull("hwchase17/openai-tools-agent")
agent = create_openai_tools_agent(llm, tools, prompt)

# 3. 创建执行器
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 4. 执行
# 问题1：需要实时信息 -> 触发搜索工具
result1 = agent_executor.invoke({"input": "今天北京天气怎么样？"})
print(result1['output'])

# 问题2：需要计算 -> 触发计算器工具
result2 = agent_executor.invoke({"input": "圆周率乘以10的平方是多少？"})
print(result2['output'])

# 问题3：结合推理与工具调用
result3 = agent_executor.invoke({"input": "OpenAI最新发布的模型是什么？它的上下文长度比GPT-4长多少？请先搜索再计算差值。"})
print(result3['output'])
```

## 知识依赖说明
理解本文内容，需要具备以下前置知识：
- **阶段4「大模型应用开发」中的「Prompt工程」**：这是LangChain的基石。你需要熟悉如何设计有效的Prompt，理解System、Human、AI等角色消息的格式，以及如何通过Prompt引导模型行为。LangChain中的`PromptTemplate`是对Prompt工程的最佳实践封装。
- **大语言模型（LLM）基础概念**：了解LLM的基本原理、输入输出格式（如ChatML）、Token概念以及主流API（如OpenAI API）的调用方式。
- **基本的Python编程**：能够理解函数、类、字典等数据结构，以及简单的API调用。

如果你对Prompt工程尚不熟悉，建议先系统学习如何构造清晰、具体、带有示例的指令（Instruction）和上下文（Context），这是构建高效、可靠LangChain应用的前提。

## 实践建议
1.  **从`LLMChain`和`ConversationBufferMemory`开始**：不要一开始就尝试构建复杂的Agent。先用一个简单的对话链结合基础记忆，感受信息在链中的流动和记忆的存储与读取过程。
2.  **为每个工具编写清晰、准确的`description`**：Agent依赖LLM根据工具描述来决定是否及如何调用工具。模糊的描述会导致错误的工具调用。描述应明确说明工具的用途、输入格式和适用场景。
3.  **在开发过程中始终开启`verbose=True`**：这能让你清晰地看到Chain或Agent内部的执行步骤、Prompt的构建过程以及工具的调用情况，是调试和理解系统行为的利器。
4.  **谨慎处理用户输入和工具输出**：在将用户输入填充到Prompt或传递给工具执行前，应考虑进行必要的清洗和验证，防止Prompt注入攻击或工具滥用。对于计算类工具，避免使用不安全的`eval`函数。
5.  **对复杂Chain进行模块化设计和测试**：将大的工作流拆分成多个职责单一的小Chain，并分别进行单元测试。确保每个小Chain在给定正确输入时能产生预期输出，然后再将它们组合起来。
